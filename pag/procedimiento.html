SOMOS EQUIPO NÓMINA

Aprende de funciones almacenadas / store procedure, tipos de datos, y mucho más...

      Un stored procedure (procedimiento almacenado) en una base de datos es un conjunto de instrucciones SQL predefinidas que se guardan en el servidor de la base de datos y se pueden llamar desde una aplicación o desde otras partes de la base de datos.

      Los stored procedures son útiles para mejorar la eficiencia y la seguridad de la base de datos, ya que se pueden llamar varias veces sin tener que volver a escribir el código cada vez, lo que reduce el tiempo de ejecución y minimiza los errores de escritura. Además, los stored procedures también pueden incluir lógica de negocio compleja y funciones que no están disponibles en el lenguaje SQL estándar.

      Los stored procedures se pueden crear utilizando lenguajes de programación como T-SQL, PL/SQL o PL/pgSQL, dependiendo del sistema de gestión de base de datos utilizado. Para utilizar un stored procedure, es necesario llamarlo desde una aplicación o desde otra parte de la base de datos mediante una sintaxis específica.

TIPOS DE DATOS

    Mientras se crea la tabla, para cada columna, especifica un tipo de dato, es decir, qué tipo de datos desea almacenar en los campos de la tabla. Algunos de estos son los de tipo númerico, los de tipo monetario, los de tipo carácteres (texto), los tipos de datos binarios, tipos de fecha/hora, los de tipo ennumerado, los de tipo geometrico,los de tipo array/arreglo, los de tipo compuesto, entre otros...

FUNCIONES

    Las funciones de PLPGSQL, también conocidas como procedimientos almacenados, le permiten realizar operaciones que normalmente requerirían varias consultas y viajes de ida y vuelta en una sola función dentro de la base de datos. Las funciones permiten la reutilización de la base de datos, ya que otras aplicaciones pueden interactuar directamente con sus procedimientos almacenados en lugar de un código de duplicación o de nivel medio.

   Ejemplo: La sintaxis básica para crear una función es la siguiente:

  • CREATE [OR REPLACE] FUNCTION function_name (arguments)
    RETURNS return_datatype AS $variable_name$
    DECLARE
    declaration;
    [...]
    BEGIN
    < function_body >
    [...]
    RETURN { variable_name | value }
    END; LANGUAGE plpgsql;
Donde:
  • -"function-name" especifica el nombre de la función.
  • -La opción "[OR REPLACE]" deja modificar una función existente.
  • -La función debe contener una declaración de retorno.
  • -La cláusula "RETURN" especifica el tipo de datos que va a devolver de la función. El return_datatype puede ser un tipo base, compuesto o de dominio, o puede hacer referencia al tipo de una columna de tabla.
  • -"function-body" contiene la parte ejecutable.
  • -La palabra clave "AS" se usa para crear una función independiente.
  • -"plpgsql" es el nombre del lenguaje en que la función está implementada. Aquí, se usa esta opción para PostgreSQL, pero también se puede usar en SQL, #C, o el nombre de un lenguaje procedural definido por el usuario. Para retrocompatibilidad, el nombre puede estar encerrado entre comillas simples.
